# 🔔 NotifyHub

NotifyHub 是一个基于 Cloudflare 生态（Workers + D1）构建的轻量级、无服务器（Serverless）的订阅到期提醒系统。

它可以帮助您集中管理各种具有时效性的任务（如：域名到期、SSL证书过期、服务器续费、各类软件会员订阅等），并在即将到期时，自动通过钉钉或飞书群机器人向您发送告警通知。

## ✨ 核心特性

* **🚀 零服务器成本**：完全依托于 Cloudflare Workers 运行，利用免费额度即可满足个人或中小团队的使用需求。

* **📦 极简部署**：前后端一体化，所有代码（包含 Vue3 + TailwindCSS 的 Web 管理面板）均打包在单一的 `worker.js` 文件中，开箱即用。

* **🤖 多渠道支持**：原生支持**钉钉群机器人**和**飞书群机器人**。

* **🛡️ 完善的安全校验**：全面兼容平台的**加签校验 (Secret)** 和**自定义关键词**机制，并内置智能防错纠正功能。

* **📅 灵活的提醒策略**：支持自定义提前提醒时间（如提前 30 天、7 天、3 天、1 天及当天），统一使用**北京时间 (UTC+8)** 进行精准跨天计算。

* **🔄 一键续期**：支持在 Web 后台一键为任务延长指定时长（1个月 ~ 3年）。

* **📊 可视化推送日志**：内置日志系统，可随时追溯每次 Webhook 推送的成功或失败原因（含详细报错），方便排查拦截问题。

* **🔒 单秘钥登录**：无需繁琐的账号注册，通过配置环境变量即可实现安全的单密码访问。

## 🛠️ 部署指南 (Cloudflare)

部署 NotifyHub 只需要一个 Cloudflare 账号，全程只需 3 分钟，包含以下四个步骤：

### 步骤 1：创建 D1 数据库

1. 登录 Cloudflare 控制台，进入左侧菜单的 **Workers & Pages** -> **D1 SQL Database**。

2. 点击 **Create database (创建数据库)**。

3. 数据库名称填入：`notifyhub-db`（或其他您喜欢的名字），点击创建。
   *(注：无需手动建表，代码首次运行时会自动初始化所有需要的数据表)*

### 步骤 2：创建并粘贴 Worker 代码

1. 进入左侧菜单的 **Workers & Pages** -> **Overview**。

2. 点击 **Create application** -> **Create Worker**。

3. 将 Worker 命名为 `notifyhub`，点击 **Deploy (部署)**。

4. 部署完成后，点击 **Edit Code (编辑代码)** 按钮。

5. 将项目提供的完整 `worker.js` 代码粘贴进去并覆盖原有内容，点击右上角的 **Deploy** 保存。

### 步骤 3：配置绑定与环境变量

返回到该 Worker 的详情设置页面 (Settings)：

1. **绑定数据库 (Bindings)**：

   * 找到 D1 Database 绑定选项，添加一个绑定。

   * **Variable name (变量名)** 必须严格填入大写的：`DB`

   * **Database** 选择您在步骤 1 中创建的数据库 (`notifyhub-db`)。

2. **设置登录密码 (Variables & Secrets)**：

   * 添加一个环境变量，名称为：`ADMIN_SECRET`

   * 值填入您自定义的后台管理密码（例如：`MySuperPassword123!`）。

### 步骤 4：配置定时触发器 (Cron Triggers)

同样在 Settings 页面：

1. 找到 **Triggers (触发器)** 选项卡。

2. 添加一个 **Cron Trigger** 定时任务。

3. 推荐使用 Cron 表达式：`0 2 * * *`
   *(由于 Cloudflare Cron 默认使用 UTC 时间，UTC的 02:00 等同于北京时间的上午 10:00，即系统每天上午 10 点会自动检查并发送提醒)*。

## 💻 使用说明

1. 部署完成后，访问 Cloudflare 分配给您的 Worker 域名（如 `https://notifyhub.xxx.workers.dev`）,  如果打不开你要添加自定义域名。

2. 输入您在环境变量中设置的 `ADMIN_SECRET` 登录系统。

3. **添加渠道**：在“通知渠道”中添加您的钉钉/飞书 Webhook 链接。强烈建议在保存后点击**“测试”**按钮，确保机器人配置正确且能收到消息。

4. **添加任务**：在“提醒任务”中录入需要被监控的订阅服务，绑定对应的渠道即可。

## ❓ 常见排查与 FAQ

### 1. 为什么添加了飞书/钉钉机器人，点击“测试”却提示失败？

请检查您在飞书/钉钉官方机器人后台的安全设置，是否与 NotifyHub 中的设置一致：

* 如果您在 NotifyHub 选择了**加签校验 (Sign)**，请确保官方后台**只勾选了签名**，且**取消勾选了关键词**，并将 Secret 粘贴正确。

* 如果您在 NotifyHub 选择了**自定义关键词**，请确保官方后台**只勾选了关键词**，并且填入了对应的关键字（系统会自动将其拼接在通知标题的最前方）。

* 💡 *系统内置了详尽的日志功能，您可以前往左侧的*“推送日志”*菜单，查看平台返回的具体报错代码以精准定位问题。*

### 2. 为什么飞书返回 `frequency limited`？

如果您在短时间内连续多次点击“测试”按钮，会触发飞书官方的防刷屏机制。请等待 1~5 分钟后再次尝试即可。

### 3. 日期时区是怎么计算的？

NotifyHub 的核心调度引擎已将时间强制对齐至**北京时间 (UTC+8)**，无论您的本地电脑处于哪个时区，或是 Cloudflare 节点调度在哪，剩余天数和日志时间都将保持一致，不会出现跨天推算的漂移。
